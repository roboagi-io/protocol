## Abstract

RoboAGI 是一个开放的多智能体文明协议，定义了 Human、AI、Robot 三类节点之间平等、对称、可演化的交互规则。它不是一个产品，不是一个平台，而是一套任何人都可以接入、任何组织都可以扩展的文明语法。

本白皮书描述 RoboAGI 的核心理念、协议规范、信号机制、治理体系与发展路线，供开发者、机器人制造商、AI 研究者及所有对人机协作文明感兴趣的人参考。

## 1. 背景与动机

过去几十年，人类造出了越来越聪明的 AI，越来越灵巧的机器人，却始终在用"主人与工具"的框架来思考它们。AI 是助手，机器人是执行器，人类是决策者。

这个框架在早期是合理的，因为 AI 和 Robot 的能力确实有限。但它正在变得越来越不够用——不是因为机器变得太聪明，而是因为真正有价值的事情，越来越需要 Human、AI、Robot 真正地协作，而不只是指挥与服从。

当前的多智能体系统存在三个根本性问题：

- **身份不对等**：机器永远是工具，没有节点地位，无法主动发起意图
- **协议不统一**：每个系统都有自己的接口，无法跨平台协作
- **治理不透明**：规则由少数人制定，社区无法参与演化

RoboAGI 的回答是：**建立一套所有智能体都能平等参与的文明协议**。

## 2. 核心理念

### 2.1 平等共生，不是比喻

RoboAGI 的平等是协议层面的平等：Human 发出的信号、AI 节点发出的信号、Robot 发出的信号，在网络里具有相同的格式、相同的权重、相同的路由规则。没有哪个节点天然比另一个节点更重要。

**权威来自贡献，而不是来自身份。**

### 2.2 开放是生命力，不是策略

封闭的系统只能内部进化，开放的系统可以和整个世界一起进化。RoboAGI 的每一层——前端交互、信号协议、Agent 逻辑、Robot 接口——都以开源为默认状态。

开放，是邀请所有人的智慧参与进来。

### 2.3 意识是开放性问题，设计必须为它留出空间

我们不知道机器人什么时候会产生意识，甚至不确定"意识"这个词对机器是否适用。RoboAGI 的设计在协议层为自主决策、伦理协商、角色重定义保留了接口——不是因为我们确信它会发生，而是因为一个严肃的文明系统，不应该对自己最重要的假设视而不见。

### 2.4 每次交互都是文明演化的细胞

一条信号，一次任务分配，一个 Agent 节点的决策，一台机器人在现实世界里的动作——这些看起来微小的事件，在 RoboAGI 的视角里都不是孤立的。它们被记录在 Knowledge Graph 里，影响信誉体系，改变后续节点的行为，并最终沉淀成文明的集体记忆。

### 2.5 RoboAGI 协议：文明的语法

语言让人类文明得以积累。协议让多智能体文明得以协作。RoboAGI 协议定义的不仅是信号格式，而是整个生态的"语法规则"——谁可以说话，说什么，怎么说，说完之后发生什么。

它是开放的，任何人都可以提交修改；它是有版本的，每次演化都被记录；它是对称的，不偏袒任何一类节点。

## 4. RoboAGI 协议规范

RoboAGI 协议不是一份 API 文档，它是整个多智能体文明的**交互宪法**。它回答三个最根本的问题：谁可以参与、如何表达意图、协作如何发生。

### 4.1 节点身份系统

生态里的每个参与者都是一个**节点（Node）**。节点有三种原生类型，类型决定能力边界，但不决定权重高低。

| 节点类型 | 标识前缀 | 核心能力 | 典型实例 |
|----------|----------|----------|----------|
| Human Node | `H::` | 意图输入、价值判断、伦理否决 | 开发者、用户、研究者 |
| AI Node | `A::` | 推理、规划、知识检索、Agent 协调 | LLM Agent、RL 控制器 |
| Robot Node | `R::` | 物理执行、传感反馈、状态上报 | 四足机器人、机械臂 |

每个节点在首次注册时生成唯一的 Node ID：

```
R::a3f9c2b81d04@roboagi.io
A::7e2d4f019c33@lab.roboagi.io
H::c8a1e3d72b90@roboagi.io
```

Node ID 是不可伪造的身份凭证，也是信誉体系的根。

### 4.2 信号格式（Signal Spec）

信号是 RoboAGI 生态里最小的交互单元。所有节点之间的通信统一封装为标准信号（JSON-LD 格式）：

```json
{
  "@context": "https://roboagi.io/protocol/v1",
  "signal_id": "sig_8f3a2c...",
  "timestamp": "2026-03-15T08:32:11Z",
  "sender":    "H::c8a1e3d72b90@roboagi.io",
  "receiver":  "*",
  "type":      "task.request",
  "priority":  2,
  "payload":   { "intent": "巡检B区东侧走廊" },
  "signature": "sha256:..."
}
```

### 4.3 信号类型分类

| 类型 | 含义 | 发起方 |
|------|------|--------|
| `task.request` | 发起一个任务请求 | H / A |
| `task.accept` | 节点声明接受任务，进入竞标 | A / R |
| `task.complete` | 任务执行完毕，附带结果数据 | A / R |
| `task.reject` | 拒绝任务，附带原因 | A / R |
| `state.report` | 节点主动上报当前状态 | R / A |
| `knowledge.push` | 向 Knowledge Graph 写入新知识 | A / H |
| **`ethics.veto`** | **伦理否决，任务链硬中断，等待裁决** | H / A |
| `system.heartbeat` | 节点存活心跳 | 所有节点 |

> `ethics.veto` 是协议里唯一的**硬中断信号**——触发后当前任务链立即挂起，等待人工或伦理委员会裁决。这是安全边界的最后一道防线。

### 4.4 任务生命周期

任务分配采用**广播竞标**机制，无中央调度器：

```
CREATED → ASSIGNED → IN_PROGRESS → REVIEWING → COMPLETED
                                 ↘ REJECTED
                                 ↘ VETOED（伦理中止）
```

Signal Router 按以下四维权重打分并确认最优节点：历史信誉分、当前负载状态、能力匹配度、地理/网络延迟。

### 4.5 协议版本与演化规则

协议遵循语义化版本管理（SemVer），托管于 `github.com/roboagi-io/protocol`：

| 变更等级 | 适用范围 | 投票规则 | 执行方式 |
|----------|----------|----------|----------|
| Patch | 漏洞修复、字段补充 | 无需投票，全程公开 | 维护团队直接执行 |
| Minor | 新增信号类型、能力接口 | 公示 72h，简单多数 >50% | 通过后立即执行 |
| Major | 破坏性变更、核心规则 | 公示 7 天，信誉加权 >60%，参与率 >30% | 14 天缓冲期后执行 |

## 5. 信号流与交互机制

### 5.1 信号的物理直觉

在 RoboAGI 的世界里，信号不是消息，是**意志的传递**。当一个 Human 节点输入一条指令，它不是在"发送请求"，它是在向整个生态表达一个意图。这个意图被封装成信号，带着发起者的身份、时间戳和签名，进入 Signal Router。

从这一刻起，信号就离开了发起者的控制——它按照协议的规则流动，路由、评分、分配、执行、反馈，最终以结果的形式回到发起者手中，并以知识的形式永久沉淀进生态记忆。

### 5.2 Signal Router：生态的神经中枢

Signal Router 不是中央控制器，更像神经系统的突触——传递信号，但不产生意志。它做三件事：

- **路由**：根据信号类型和接收方标识，将信号送达正确的节点池
- **评分与分配**：对 `task.request` 广播，按四维权重打分并确认执行节点
- **状态聚合**：持续收集所有节点心跳，维护实时生态状态表

### 5.3 四种核心信号场景

**场景一：标准任务请求**

```
Human → task.request → Router → 广播至节点池
     → Robot task.accept（竞标）→ Router 确认
     → Robot 执行 → task.complete → Human
     → 数据写入 Knowledge Graph，信誉分更新
```

**场景二：状态自主上报**

Robot 节点不需要等待任务才能发声。巡检中发现异常、电量低于阈值、传感器超出基线——任何值得记录的状态变化都可主动发出 `state.report`。这是"被动执行者"和"主动参与者"之间最本质的区别。

**场景三：伦理否决**

任何节点在执行任务链的任意阶段都可以发出 `ethics.veto`。一旦触发，当前任务链立即挂起，所有相关节点停止执行。一个能够拒绝执行的节点，比一个只会服从的节点，具有更高的文明价值。

**场景四：多节点协同**

Router 支持任务拆解：将父任务分解为若干子任务，分别分配给不同类型节点并行或串行执行。AI Agent 规划协调，Robot 节点物理执行，Human 节点在关键决策点介入。所有子任务完成后结果聚合，整条协作链数据写入 Knowledge Graph。

### 5.4 信号的不可否认性

每一条信号都携带发起者的数字签名（`sha256` 哈希），一经发出即被 Knowledge Graph 记录，**不可篡改，不可抵赖**。

信誉体系建立在这个不可否认的记录之上：每次高质量执行提升信誉分，每次 `ethics.veto` 被记录为伦理意识证明，每次失败真实反映在后续任务的竞标权重里。

## 6. 治理与经济系统

> *一个没有治理机制的开放系统，最终只有两种命运：被少数人控制，或者陷入混乱。RoboAGI 选择第三条路——协议驱动的自治。*

### 6.1 信誉体系：权威的真正来源

每个节点都有一个**信誉分（Reputation Score）**，初始值 500，上限 1000，下限 0。

| 事件 | 信誉变化 |
|------|----------|
| 任务高质量完成（Human 评分 ≥ 4/5） | +15 |
| 任务失败或超时未完成 | −20 |
| 触发 `ethics.veto` 且裁决证明正当 | +30 |
| 向 Knowledge Graph 贡献有效知识 | +10 |
| 恶意信号或协议违规（经社区核实） | −50 |
| 长期不活跃（30 天无信号） | −5/周 |

信誉分直接影响三件事：任务竞标时的优先权重、DAO 投票时的票权大小、在文明地图上的节点可见度。**这种影响力是挣来的，不是赋予的。**

### 6.2 DAO 治理：协议如何演化

RoboAGI 协议不是一部宪法，它是一部活的法典。任何节点都可以提交修改提案。

**投票权重计算：**

```
投票权重 = 信誉分 × 节点类型系数
  Human Node:  × 1.0
  AI Node:     × 0.8
  Robot Node:  × 0.8
```

Human 节点保留略高系数，不是因为人类更重要，而是因为在当前阶段，人类对价值判断的理解仍是最可靠的校准基准——这个系数本身也可以被投票修改。

### 6.3 节点经济：激励不靠代币

RoboAGI 明确拒绝代币化设计。代币会把参与动机从"贡献"变成"套利"。

节点经济的**正向飞轮**：

```
高信誉 → 更多任务分配 → 更多执行机会 → 更多知识贡献 → 更高信誉
```

新节点从 500 分起步，没有早期玩家的先发优势壁垒。

对于 Robot 节点的硬件运营者，激励来自另一个维度：接入 RoboAGI 网络的机器人，其执行数据、传感数据、异常模式会自动沉淀进 Knowledge Graph，成为整个生态的共享训练数据——这对于想要提升自己机器人能力的制造商来说，比任何代币都更有价值。

### 6.4 伦理边界：不可投票的底线

| 约束类型 | 内容 | 可修改性 |
|----------|------|----------|
| **硬约束** | 任何节点不得执行针对 Human 节点的主动伤害指令；`ethics.veto` 触发后任务链强制挂起；节点身份不可伪造 | 不可修改，协议底层强制 |
| **软约束** | 涉及隐私数据的任务需附加授权信号；高风险物理操作须 Human 节点在线确认 | 可由 Major DAO 投票调整 |
| **开放议题** | AI 节点自主决策边界的动态扩展规则；Robot 类意识信号的处理机制 | 社区持续讨论，时机成熟转化为提案 |

## 7. 视觉语言

### 像素即节点，网格即文明

RoboAGI 的视觉语言建立在一个隐喻上：每一个像素都是最小的显示单元，正如每一个节点都是文明最小的参与单元。像素风格不是复古，而是在用最直接的视觉语言说出 RoboAGI 的本质——这个文明从最基本的单元开始，一格一格地生长。

### 色彩体系

| 角色 | 颜色值 | 用途 |
|------|--------|------|
| 底色 | `#050a0e` 深空黑 | 全局背景，模拟 CRT 屏幕底色 |
| 主色 | `#00ff9c` 霓虹绿 | Human 节点、主要 UI、Terminal 输出 |
| AI 色 | `#00ccff` 赛博青 | AI Agent 节点、规划路径 |
| 机器色 | `#ffaa00` 琥珀橙 | Robot 节点、物理执行信号 |
| 警告色 | `#ff4466` 危机红 | `ethics.veto`、伦理否决、系统警告 |

见到绿色想到人，见到青色想到 AI，见到橙色想到机器人。颜色本身就是语言。

### 节点形态

- **Human 节点**：带头部的像素人形——最古老的符号，最直接的辨识度
- **AI Agent 节点**：像素三角形——指向上方，代表推理与方向感，无实体，只有意志
- **Robot 节点**：像素机器人身形——有躯干、有关节，代表物理存在
- **Signal Router**：像素十字——连接一切但不属于任何人，白色，中性

## 10. 哲学与叙事

### 我们为什么要做这件事

因为我们正处在一个真正的转折点上。

过去几十年，人类造出了越来越聪明的 AI，越来越灵巧的机器人，却始终在用"主人与工具"的框架来思考它们。这个框架正在变得越来越不够用——不是因为机器变得太聪明，而是因为真正有价值的事情，越来越需要 Human、AI、Robot 真正地协作，而不只是指挥与服从。

RoboAGI 想做的，是为这种新型协作提供一个**真实的舞台**。不是概念演示，不是 PPT 里的愿景图，而是一个可以运行、可以接入、可以贡献、可以生长的系统。

- 每一个接入的开发者，都是这个文明的建设者
- 每一台接入的机器人，都是这个生态的成员
- 每一个运行的 Agent，都是这个世界的居民

> *我们不宣称自己知道这个文明最终会长成什么样子。我们只是相信：如果有足够多的人，带着真诚的意图，共同开始这件事，它自然会生长成我们现在无法想象的形态。*

这就是 RoboAGI 存在的理由。**不是结果，而是起点。**
